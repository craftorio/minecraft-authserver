# AGENTS.md — rules for AI agents

This file describes how agents should work with the `minecraft-authserver` repository.

## Mandatory rules

### English only (strict)

All project text must be written in **English**:

- README, AGENTS.md, CLAUDE.md, and any other documentation
- Code comments and docblocks
- Commit messages, PR descriptions, and issue text
- Error messages and log strings added by agents

Do not add Russian or other non-English text unless the user explicitly requests a translation.

### No agent attribution

**Never** add mentions of yourself, AI, code-generation tools, or automation in:

- commit messages and git trailers (`Co-authored-by`, `Signed-off-by` on behalf of an agent, etc.)
- pull request and issue descriptions
- README, AGENTS.md, CLAUDE.md, and other documentation
- code comments
- changelog, release notes, or logs

Forbidden patterns (non-exhaustive):

- "Generated by …", "Written by AI", "AI-assisted"
- "Co-authored-by: Cursor / Claude / Copilot / ChatGPT …"
- references to Cursor, Claude, Copilot, ChatGPT, OpenAI, Anthropic as authors of changes

Commits and documentation should read as if written by a human project developer.

### Code comments

Add **detailed English comments** wherever logic is non-obvious:

- protocol compatibility (Yggdrasil/Mojang conventions)
- UUID/id derivation, endianness, hashing schemes
- hybrid storage (MySQL + SleekDB), lazy migration
- security-sensitive paths (session validation, texture signing)
- intentional deviations from spec (e.g. disabled `serverId` filter)

Do not comment obvious code (getters, simple CRUD). Match existing comment style.

### Git

- Create commits **only when explicitly asked** by the user.
- Do not run destructive git operations without an explicit request.
- Do not push to remote without an explicit request.

### Scope of changes

- Minimal diff: only what the task requires.
- Do not refactor or "improve" code outside scope.
- Follow project conventions (PSR-4, `declare(strict_types=1)`, Flight routes, PHP-DI).

### Documentation

- Do not create or expand markdown files unless the user asks.
- Exception: the task explicitly requests documentation updates.

### Tests

- Do not add tests unless requested or they provide meaningful coverage.

## Project context

- **Language:** PHP 7.3
- **Namespace:** `Craftorio\Authserver\`
- **Web:** Flight micro-framework, entrypoint `authserver/public/index.php`
- **CLI:** Symfony Console, entrypoint `authserver/bin/console`
- **DI:** PHP-DI, configured in `Authserver::configureDI()`
- **Storage:** MySQL (`Account\Storage\Mysql`) or SleekDB (`Account\Storage\SleekDb`)
- **Config:** `authserver/config.php` (from `config.example.mysql.php`)

## Common tasks

### Run and debug

```bash
docker compose up -d
docker compose exec cli php bin/console list
curl http://localhost:8187/
```

### New HTTP route

1. Create a class in `authserver/src/Route/` implementing `RouteInterface`.
2. Register it in the `$routes` array in `Authserver.php`.
3. Inject dependencies via constructor (PHP-DI autowire).

### New CLI command

1. Create a class in `authserver/src/Command/`.
2. Register it in `authserver/bin/console`.

## Directory layout

```
authserver/src/
  Account/Storage/   # Mysql, SleekDb
  Authenticator/     # Authentication
  Command/           # CLI commands
  Entity/            # Account, Profile, Skin
  Hash/              # Phpass, Md5
  Route/             # HTTP routes
docker/              # nginx, php-fpm
```

## Avoid

- Major PHP/dependency upgrades without request.
- Storing secrets in the repository.
- Changing `docker-compose.yml` volume paths without understanding prod layout (`/var/www/launcher/MinecraftSkins`).
